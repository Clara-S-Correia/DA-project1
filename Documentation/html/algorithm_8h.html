Detailed Description

Implementation of the Edmonds-Karp algorithm for Maximum Flow.

This file contains the logic for finding augmenting paths via BFS and updating the residual graph to compute the optimal matching between submissions and reviewers.

Finds an augmenting path in the residual graph using Breadth-First Search.

Explores the graph starting from the source to find the shortest path (by number of edges) to the sink that still has available residual capacity (Capacity - Flow > 0).

  • Template Parameters
    TThe type of data stored in the vertices (typically int).
    Parameters
    gPointer to the Graph object.
    sPointer to the source Vertex.
    tPointer to the sink Vertex.
    Returns
    true If a path to the sink was found.
    false If no more augmenting paths exist.
  • @time_complexity O(V + E) where V is the number of vertices and E is the number of edges.

Computes the Maximum Flow using the Edmonds-Karp algorithm.

Iteratively calls BFS_part to find augmenting paths. For each path, it identifies the "bottleneck" capacity and updates the flow along the forward and reverse edges.

  • Template Parameters
    TThe type of data stored in the vertices.
    Parameters
    gPointer to the Graph object.
    sourceThe identifier for the source node (usually 0).
    sinkThe identifier for the sink node (usually numSub + numRev + 1).
  • @time_complexity O(V * E^2). In a flow network, this ensures the algorithm terminates in polynomial time regardless of the capacity values. @space_complexity O(V) for the queue and path tracking used in BFS.
